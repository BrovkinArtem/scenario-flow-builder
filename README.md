# Конструктор сценариев

Прототип CRM-модуля: **список сценариев** и **редактор блок-схемы** (React Flow). Состояние графа — **Zustand**; данные — **мок-API** поверх `localStorage` с задержкой **300–500 ms**. Добавлена **локализация** (английский и русский) сверх требований ТЗ.

---

## Запуск

Нужны **Node.js 18+** и npm.

```bash
npm install
npm run dev
```

Откройте адрес из вывода Vite (обычно `http://localhost:5173`).

```bash
npm run build   # проверка TypeScript + production-сборка
npm run preview # локальный просмотр dist
```

### Сброс мок-данных

В DevTools → Application → Local Storage удалите ключ `scenario-flow-builder:scenarios` и перезагрузите страницу.

### Опционально: имитация ошибки сохранения

Файл `.env.local`:

```bash
VITE_SIMULATE_SAVE_ERROR=true
```

## Соответствие тестовому заданию (чеклист)

| Требование | Реализация |
|------------|------------|
| Список сценариев: название, дата обновления | `ScenarioList`, формат даты с учётом языка |
| Кнопка «Создать новый» | В RU: **«Создать новый»** (`createScenario` в `shared/config/i18n.ts`); создание через `saveScenario` + переход в редактор |
| Мок + `localStorage` + задержка 300–500 ms | `shared/lib/delay.ts`, `entities/scenario/api/mockScenarioApi.ts` |
| Холст (flow) | `widgets/flow-canvas/FlowCanvas.tsx`, `@xyflow/react` |
| Узлы «Действие» / «Условие» | Свои ноды + панель добавления |
| Соединение узлов | `onConnect` + рёбра; валидация дубликатов и петель |
| Перетаскивание узлов | Стандартное поведение React Flow |
| Удаление узла и связи | Backspace/Delete, кнопка «Удалить выбранное»; при удалении узла **снимаются все связанные рёбра** |
| Сохранение: узлы и рёбра + имитация запроса | `useSaveScenario` → `saveScenario()` с задержкой |
| Загрузка и ошибки для списка | `useScenariosList` + интерфейс |
| Загрузка и ошибки для сохранения | `useSaveScenario` + уведомления в панели |
| Загрузка и ошибки при открытии сценария | `ScenarioEditorPage`: спиннер, `notFound`, ошибка с **повтором запроса** |
| Строгий TypeScript, сущности узел / связь / сценарий | `entities/flow/model/types.ts`, `entities/scenario/model/types.ts` |
| Декомпозиция интерфейса, стили | Tailwind, слои `shared/ui`, `widgets`, `features`, `pages` |

## Стек и архитектура

| Технология | Назначение |
|------------|------------|
| **Vite** | Сборка и dev-сервер |
| **React 18** | Интерфейс |
| **TypeScript (strict)** | Типизация |
| **React Router** | Маршруты: список / редактор |
| **Tailwind CSS** | Стили |
| **Zustand** | Состояние графа (узлы, рёбра, обработчики React Flow) |
| **@xyflow/react** | Редактор flow |
| **i18next + react-i18next** | Локализация (сверх ТЗ) |

Структура каталогов — **по фичам** (`feature-sliced`): `app` (оболочка, роутер), `pages`, `widgets`, `features`, `entities`, `shared`. Данные сценариев — `entities/scenario/api` (мок); интерфейс обращается к ним через хуки страниц и фич.

### Использование ИИ

Инструменты на базе ИИ использовались для:

- первичного каркаса проекта;
- структуры и текста README;
- мелкого шаблонного кода.

Весь сгенерированный код:

- просмотрен вручную;
- при необходимости отрефакторен;
- проверен на соответствие поведению и архитектуре (в том числе `npm run build` и дымовые проверки: список, редактор, сохранение).

**Основная логика, архитектурные решения и итоговая структура репозитория** — под контролем и ответственностью разработчика.

## Дерево проекта (кратко)

```
src/
  app/          — вёрстка оболочки, роутер, провайдеры
  pages/        — список сценариев, редактор
  widgets/      — список, холст, ноды
  features/     — создание, сохранение, операции на холсте
  entities/     — сценарий и flow (типы, мок-API, store)
  shared/       — общий UI, хуки, конфиг (в т.ч. i18n)
```
